# Phase 5: Visual Captioning & Table of Contents (ToC)

## Overview
In this phase, we ingest the JSON array of event triggers generated by the Phase 4 Sentry. For every timestamp identified as an "Interest Zone", we extract the corresponding visual frame from the source video. We then pass this frame through **Qwen2.5-VL-3B-Instruct (4-bit)** (via MLX) to generate a dense semantic caption of the user's action. The output is a highly structured Table of Contents (ToC) that **merges** acoustic trigger captions with the Phase 2 event annotations to ensure full temporal coverage.

## Optimization for Apple Silicon
To maintain speed without sacrificing capability, we use the `mlx-vlm` library to run **Qwen2.5-VL-3B-Instruct-4bit** (~2 GB 4-bit weights). The models should be cached on the 2TB SSD to avoid clogging the internal M4 Mac storage.

> [!NOTE]
> **Caption Quality Consideration:** Qwen2.5-VL-3B (~3B params, 4-bit) offers strong procedural action understanding. If caption quality proves insufficient during validation, consider upgrading to a quantized Qwen2.5-VL-7B (via `mlx-vlm`) for a subset of critical triggers, or using multi-frame context.

## Trigger-Aware Prompting
Different trigger types imply different temporal contexts. We adapt the VLM prompt accordingly:

| Trigger Type | Prompt Variant |
|---|---|
| `spike` (action start) | "What action is the user starting to perform with their hands? Be specific about tools and objects." |
| `drop` (action end) | "What action did the user just finish performing? Describe the result of the completed action." |
| `forced_silence` | "What is the user currently doing? Describe the ongoing activity and any objects being manipulated." |

## Pseudocode Implementation

```python
import cv2
import json
import os
import gc
from mlx_vlm import load, generate
from mlx_vlm.prompt_utils import apply_chat_template
from mlx_vlm.utils import load_config

# System Configuration
SSD_BASE = "/Volumes/Extreme SSD/ego4d_data"
MODEL_ID = "mlx-community/Qwen2.5-VL-3B-Instruct-4bit"

# Trigger-aware prompts for better caption relevance
PROMPTS = {
    "spike": "What action is the user starting to perform with their hands? Be specific about tools and objects.",
    "drop": "What action did the user just finish performing? Describe the result of the completed action.",
    "forced_silence": "What is the user currently doing? Describe the ongoing activity and any objects being manipulated.",
    "event_sample": "Describe exactly what the person is doing in this frame. Be specific about the objects being used and the action being performed. Answer in one sentence."
}

class CaptioningEngine:
    def __init__(self):
        # Load Qwen2.5-VL-3B using MLX optimizations
        print("Loading Qwen2.5-VL-3B-Instruct-4bit into unified memory...")
        os.environ['HF_HOME'] = os.path.join(SSD_BASE, "models/huggingface")
        self.model, self.processor = load(MODEL_ID)
        self.config = load_config(MODEL_ID)

    def unload(self):
        """Free model memory before next phase."""
        del self.model
        del self.processor
        gc.collect()
        print("Captioning engine VLM unloaded.")

    def extract_frame(self, video_path: str, timestamp_s: float, video_uid: str) -> str:
        """ 
        Extracts a specific frame and saves it to the SSD with a unique filename.
        Returns the path to the saved image.
        """
        cap = cv2.VideoCapture(video_path)
        
        # Seek to the requested timestamp (in milliseconds)
        cap.set(cv2.CAP_PROP_POS_MSEC, timestamp_s * 1000)
        ret, frame = cap.read()
        cap.release()
        
        if not ret:
            raise ValueError(f"Could not read frame at timestamp {timestamp_s}")
        
        # Use unique filename per (video, timestamp) to avoid race conditions
        safe_ts = str(timestamp_s).replace('.', '_')
        img_path = os.path.join(SSD_BASE, f"cache/phase5/frames/{video_uid}_{safe_ts}.jpg")
        os.makedirs(os.path.dirname(img_path), exist_ok=True)
        cv2.imwrite(img_path, frame)
        return img_path
    
    def extract_multi_frame_context(self, video_path: str, timestamp_s: float, video_uid: str, window_sec: float = 0.5) -> list:
        """
        Extract 3 frames around the trigger point for richer context.
        Returns list of image paths: [before, at, after].
        """
        timestamps = [
            max(0, timestamp_s - window_sec),
            timestamp_s,
            timestamp_s + window_sec
        ]
        paths = []
        for ts in timestamps:
            try:
                path = self.extract_frame(video_path, ts, video_uid)
                paths.append(path)
            except ValueError:
                pass  # End of video, skip
        return paths

    def _caption_frame(self, img_path: str, trigger_type: str) -> str:
        """Run VLM inference on a single frame with trigger-aware prompting."""
        prompt = PROMPTS.get(trigger_type, PROMPTS["forced_silence"])
        try:
            formatted = apply_chat_template(
                self.processor,
                self.config,
                prompt,
                num_images=1
            )
            output = generate(
                self.model,
                self.processor,
                img_path,
                formatted,
                max_tokens=64,
                temperature=0.2
            )
            return output.text.strip() if hasattr(output, 'text') else str(output).strip()
        except Exception as e:
            print(f"[ERROR] VLM inference failed for {img_path}: {e}")
            return "[CAPTION_FAILED]"

    def process_video_toc(self, video_uid: str, video_path: str):
        """
        Build a Table of Contents for one video by captioning acoustic triggers
        and merging with Phase 2 event annotations.

        Args:
            video_uid: Unique identifier for the video.
            video_path: Absolute path to the video file on the SSD.
        """
        # Cache check — skip if ToC already exists
        toc_out_path = os.path.join(SSD_BASE, f"cache/phase5/{video_uid}_toc.json")
        if os.path.exists(toc_out_path):
            print(f"[CACHED] ToC for {video_uid} already exists.")
            with open(toc_out_path, 'r') as f:
                return json.load(f)

        trigger_path = os.path.join(SSD_BASE, f"cache/phase4/{video_uid}_triggers.json")
        
        if not os.path.exists(trigger_path):
            raise FileNotFoundError(f"Triggers for {video_uid} not found at {trigger_path}. Run Phase 4 first.")
            
        with open(trigger_path, 'r') as f:
            triggers = json.load(f)
            
        toc_entries = []
        os.makedirs(os.path.join(SSD_BASE, "cache/phase5/frames"), exist_ok=True)
        
        print(f"Processing {len(triggers)} triggers for {video_uid}...")
        
        for t in triggers:
            timestamp = t['timestamp']
            trigger_type = t['trigger_type']
            
            # 1. Grab visual context (primary frame)
            try:
                img_path = self.extract_frame(video_path, timestamp, video_uid)
            except ValueError as e:
                print(f"[SKIP] Frame extraction failed at {timestamp}s: {e}")
                continue
            
            # 2. Run VLM inference with trigger-aware prompt
            caption_text = self._caption_frame(img_path, trigger_type)
            
            # 3. Quality filter: skip obviously generic/failed captions
            if len(caption_text) < 5 or caption_text == "[CAPTION_FAILED]":
                print(f"[WARN] Low quality caption at {timestamp}s, retrying with default prompt...")
                caption_text = self._caption_frame(img_path, "event_sample")
                if len(caption_text) < 5 or caption_text == "[CAPTION_FAILED]":
                    caption_text = "Unable to caption frame"
            
            # 4. Construct ToC Entry
            entry = {
                "timestamp": timestamp,
                "trigger_type": trigger_type,
                "caption": caption_text
            }
            toc_entries.append(entry)
            print(f"[{timestamp}s | {trigger_type}]: {entry['caption']}")
            
        # 5. Merge with Phase 2 Event Annotations
        event_log_path = os.path.join(SSD_BASE, f"cache/phase2/{video_uid}_events.json")
        if os.path.exists(event_log_path):
            with open(event_log_path, 'r') as f:
                event_log = json.load(f)
            # Add event annotations as ToC entries with source="event_sample"
            for event in event_log:
                toc_entries.append({
                    "timestamp": event["start_time"],
                    "end_time": event["end_time"],
                    "trigger_type": "event_sample",
                    "caption": event["caption"]
                })
            print(f"  Merged {len(event_log)} event annotations from Phase 2")
        
        # Sort by timestamp for chronological ordering
        toc_entries.sort(key=lambda x: x["timestamp"])
        
        # 6. Save Final ToC
        os.makedirs(os.path.dirname(toc_out_path), exist_ok=True)
        with open(toc_out_path, 'w') as f:
            json.dump(toc_entries, f, indent=4)
        
        print(f"ToC successfully constructed for {video_uid}. ({len(toc_entries)} entries)")
        return toc_entries

if __name__ == '__main__':
    engine = CaptioningEngine()
    # Example: run on an EPIC-KITCHENS video
    engine.process_video_toc(
        "video_uid_example",
        "/Volumes/Extreme SSD/ego4d_data/v2/full_scale/video_uid_example.mp4"
    )
    engine.unload()
```

## Session Resilience

Each phase must be resumable from a different session. Phase 5's contract:

| Item | Detail |
|---|---|
| **Input Dependencies** | `cache/phase4/{video_uid}_triggers.json` (Phase 4), `cache/phase2/{video_uid}_events.json` (Phase 2) |
| **Output Artifact** | `/Volumes/Extreme SSD/ego4d_data/cache/phase5/{video_uid}_toc.json` |
| **Cache Check** | On entry, `process_video_toc()` checks if `{video_uid}_toc.json` exists and returns early if so |
| **Verification Checkpoint** | After completing all videos, write `cache/phase5/_manifest.json` listing all processed video IDs and entry counts |
| **Resume Strategy** | On re-run, skip any video whose `_toc.json` already exists on the SSD |

```python
def write_phase5_manifest(processed_videos: dict):
    """Write manifest of all processed videos for session resilience."""
    manifest = {
        "phase": 5,
        "description": "Visual Captioning & ToC Construction",
        "processed": {vid: {"entries": count, "path": f"cache/phase5/{vid}_toc.json"} 
                      for vid, count in processed_videos.items()},
        "total_videos": len(processed_videos)
    }
    manifest_path = os.path.join(SSD_BASE, "cache/phase5/_manifest.json")
    with open(manifest_path, 'w') as f:
        json.dump(manifest, f, indent=2)
    print(f"Phase 5 manifest written: {len(processed_videos)} videos")
```

## Verification Strategy
- **Spot-Check Frame I/O:** Frames are saved with unique filenames per (video, timestamp). Manually inspect 10 saved JPEGs to confirm they match the expected video moments.
- **Prompt Differentiation Test:** Process a video with known spike/drop/silence triggers. Verify that "spike" captions describe *starting* actions while "drop" captions describe *completed* actions.
- **Hallucination Rate:** Read through the generated ToC JSON. Check if Qwen2.5-VL-3B is consistently returning generic phrases (e.g. "a person's hand") vs highly descriptive steps ("user is turning a wrench"). If generic, evaluate upgrading to Qwen2.5-VL-7B for critical frames.
- **Caption Coverage:** Assert that `len(toc_entries) / len(triggers) > 0.95` — i.e., fewer than 5% of triggers fail to produce a usable caption.
- **Cache Consistency:** Run `process_video_toc()` twice for the same video. Assert the second call returns cached results instantly.
